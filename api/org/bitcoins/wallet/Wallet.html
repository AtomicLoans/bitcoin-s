  31.  def getMatchingBlocks(scripts: Vector[ScriptPubKey], startOpt: Option[BlockStamp] = None, endOpt: Option[BlockStamp] = None, batchSize: Int = 100, parallelismLevel: Int = Runtime.getRuntime.availableProcessors())(implicit ec: ExecutionContext): Future[Vector[BlockMatchingResponse]]

    Iterates over the block filters in order to find filters that match to the given addresses

    Iterates over the block filters in order to find filters that match to the given addresses

    I queries the filter database for batchSize filters a time and tries to run GolombFilter.matchesAny for each filter.

    It tries to match the filters in parallel using parallelismLevel threads. For best results use it with a separate execution context.

    scripts

    list of ScriptPubKey's to watch

    startOpt

    start point (if empty it starts with the genesis block)

    endOpt

    end point (if empty it ends with the best tip)

    batchSize

    number of filters that can be matched in one batch

    parallelismLevel

    max number of threads required to perform matching (default Runtime.getRuntime.availableProcessors())

    returns

    a list of matching block hashes
